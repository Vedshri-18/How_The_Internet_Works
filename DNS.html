DNS

(Image Credit: https://www.varonis.com/)

The Domain Name System (DNS) is the phonebook of the Internet. Humans access information online through domain names, like nytimes.com or espn.com. Web browsers interact through Internet Protocol (IP) addresses. DNS translates domain names to IP addresses so browsers can load Internet resources. Each device connected to the Internet has a unique IP address which other machines use to find the device. DNS servers eliminate the need for humans to memorize IP addresses such as 192.168.1.1 (in IPv4), or more complex newer alphanumeric IP addresses such as 2400:cb00:2048:1::c629:d7a2 (in IPv6).


HOW DOES A DNS QUERY WORK?

(Image Credit: https://totaluptime.com/)

When you open your browser and type in “www.example.com”, or a DNS client needs to look up a name used in a program, it queries DNS servers to resolve the name. Each query message the client sends contains three pieces of information, specifying a question for the DNS server to answer:

  • A specified DNS domain name, stated as a fully qualified domain name (FQDN) such as “www.example.com”
  • A specified query type, which can either specify a resource record by type or a specialized type of query operation such as A, CNAME, NS etc.
  • A specified class for the DNS domain name such as IN (Internet)

To understand the process of sending a query for a specific domain or IP and getting the record that corresponds to it, let us understand the steps involved a DNS lookup:

  1. A DNS request starts when you try to access a computer on the internet.
  2. The first stop for the DNS request is the local DNS cache.
  3. If you don’t have the IP address in your local DNS cache, DNS will check with a recursive DNS server.
  4. The recursive DNS server has its own cache, and if it has the IP address, it will return it to you.
  5. The next stop is the TLD name servers, in this case, the TLD name server for the .com addresses.
  6. What the TLD name servers do have is the location of the authoritative name server for the requested site.
  7. Your local DNS service gets the IP address and connects to www.varonis.com to download all the glorious content.

4 DNS SERVERS THAT LOAD THE WEBPAGE ARE:

  • DNS Recursor
  • Root Name Server
  • TLD Name Server
  • Authorative Name Server

TYPES OF DNS QUERIES:
